About

To me, quality software means a system where context, prompts, and engineering discipline work together.

I build products with Next.js and TypeScript, backend services with Java (Spring Boot), and data layers with PostgreSQL. For integration workflows I use n8n, and for delivery I use Docker/Podman with VPS/Dokploy.

Across my projects, I handle LLM usage with context engineering and prompt engineering practices. I also actively use tools like Cursor, Codex, and Claude Code to move faster with clearer quality control.

How I work

  • Clarify the problem, constraints, and what “done” means first
  • Small, repeatable slices of delivery with early feedback
  • Observability and the simplest practical rollback paths
  • For LLMs: context engineering, prompt engineering, and guardrail-first evaluation loops
  • For agentic coding: clear task decomposition, tool selection, and explicit quality gates
